## 第2章: 因果主張のレベルの適切性

### 2.1 因果主張の4段階

| レベル | 定義 | 表現例 |
|:------|:-----|:------|
| **レベル1: 事実** | 測定可能な観察事実 | 「売上は40万円である」 |
| **レベル2: 相関** | 2つの変数の関連性 | 「客数減少と売上減少が同時に起きている」 |
| **レベル3: 仮説** | 因果関係の推論 | 「客数減少が売上減少の一因である可能性がある」 |
| **レベル4: 因果** | 検証された因果関係 | 「客数減少が売上減少を引き起こした（RCTで検証済み）」 |

---

## 第3章: Cherry-picking（都合の良い数字の選択）の検証

### 3.1 Cherry-pickingの定義

**Cherry-picking**: 都合の良いデータや期間だけを選択し、不都合なデータを無視すること

---

## 第4章: 推定値の前提の妥当性検証

### 4.1 客単価2,750円の妥当性

#### 推定方法

```
シーシャ基本（2名以上）: ¥2,400
スタンダードドリンク: ¥500
SNS投稿割引: -¥200
合計: ¥2,700

→ 各パターンの平均として¥2,750と推定
```

#### 妥当性評価

| 項目 | 評価 | 理由 |
|:-----|:---:|:-----|
| 理論値の算出 | ○ | 論理的に計算されている |
| 実測値との比較 | ✗ | 実測データなし |
| 検証可能性 | ○ | 1ヶ月のPOSデータで検証可能 |

**問題点**: 実測値がなく、理論値のみに依存している

**影響度**: 客単価が±10%ズレた場合、推定客数が±10%ズレる

| 客単価 | 推定客数（1月） | 1日平均 |
|:------|:--------------|:-------|
| 2,500円 | 160人 | 5.3人 |
| 2,750円 | 145人 | 4.8人 |
| 3,000円 | 133人 | 4.4人 |

**推奨アクション**: 1ヶ月間、実際の客単価を記録して検証する

## 第8章: P-hacking防止の教訓

### 8.1 今回の分析から得られた教訓

#### 教訓1: 検証的分析と探索的分析の区分は必須

**良い例**: Mode-Bレポートの明示的区分

**悪い例**: 仮説を事実として断言する

---

#### 教訓2: 因果主張には慎重に

**適切な表現**:
```
技術力93%を活かせば、月商+26.5万円が見込める（仮説）
```

**不適切な表現**:
```
技術力93%は月商+26.5万円の価値がある（断定）
```

---

#### 教訓3: 推定値には幅を持たせる

**適切な表現**:
```
保守的シナリオ: +34,590円（成功率60%）
標準シナリオ: +46,120円（成功率80%）
楽観的シナリオ: +57,650円（成功率100%）
```

**不適切な表現**:
```
改善余地: +57,650円（確定値のように見える）
```

---

#### 教訓4: 実測データで仮説を検証する

**分析の流れ**:
```
1. 仮説を立てる（探索的分析）
2. 施策を実行する
3. データを測定する
4. 仮説を検証する（検証的分析）
5. 仮説を修正する or 確信度を上げる
```

## 付録: 用語集

| 用語 | 定義 |
|:-----|:-----|
| **P-hacking** | 都合の良いデータや分析方法を選んで、有意な結果を得ようとすること |
| **検証的分析** | 事前に立てた仮説を検証する分析。測定可能な事実を扱う |
| **探索的分析** | データから仮説を発見する分析。新しい気づきを得る |
| **因果主張** | AがBを引き起こすという主張 |
| **相関** | AとBが同時に起きているという観察事実 |
| **反証可能性** | 仮説が間違っている場合、それを証明できる構造 |
| **Cherry-picking** | 都合の良いデータだけを選んで提示すること |
