## 1. 設計方針

### 1.1 レポートの目的

このレポートは、技術力と経営力を統合的に診断し、**3分で現状を把握し、即座にアクションを開始できる**実務的なレポートを目指します。

### 1.2 読者ペルソナ

| 項目 | 内容 |
|:---|:---|
| 対象 | シーシャ店舗オーナー（ワンオペ経営者） |
| 読書環境 | スマホ or PC、営業前後の短時間 |
| 知識レベル | シーシャ技術は高い、経営知識は基礎レベル |
| 期待 | 「今日から何をすればいいか」が明確になること |

### 1.3 核心的制約

- **3分で理解**: エグゼクティブサマリーのみで全体像が掴めること
- **即座にアクション可能**: 72時間以内に実行できる具体策の提示
- **データ制約の明示**: 推定値と事実を明確に区別すること
- **感情に配慮**: 売上減少という厳しい現実を、希望とセットで伝えること

### 1.4 デザイン哲学

**「複雑さを隠し、行動を促す」**

- 診断スコアは見せるが、計算式は付録へ
- 技術解説は必要最小限、アクションに直結する部分のみ
- 理論<実践の優先順位を視覚的に表現

### 2.2 各章の目的

| 章 | 読者に理解させること | 読者に判断させること | 読了時間 |
|:---|:---|:---|:---:|
| エグゼクティブサマリー | 現状・スコア・優先順位 | 「今日から動く」決断 | 3分 |
| 第1章 | 売上減少の構造・上限 | 「ワンオペでどこまで伸びるか」の理解 | 5分 |
| 第2章 | 技術力は高い、微調整のみ | 技術不足が原因ではないこと | 8分 |
| 第3章 | 経営知識の盲点 | 顧客視点への転換の必要性 | 10分 |
| 第4章 | 技術→収益変換の不在 | 仕組みづくりの必要性 | 7分 |
| 第5章 | 何を、いつ、どうやるか | 明日の行動選択 | 12分 |

**合計読了時間**: 約53分（流し読みで30分、精読で70分）

### 2.3 情報の流れ

```
【認識】現状は厳しい → 【理解】でも技術は高い → 【希望】仕組みがあれば回復する
     ↓
【具体】今日からできること → 【計画】30日/90日の道筋 → 【展望】12ヶ月後の姿
```

## 4. 各章の設計

### 4.1 第1章: 現状分析

#### 目的
売上減少の構造的要因を理解し、「技術の問題ではない」ことを納得させる

#### 読者に伝えるメッセージ
「売上減少の主因は季節要因と仕組み不足。技術不足ではない」

#### 図表
1. 売上推移表（3ヶ月）
2. ワンオペ上限の試算表
3. 主因ランキング表（確信度付き）

#### 構成
```
1.1 売上推移（事実の提示）
  - 3ヶ月の数字
  - 1日平均客数の推定

1.2 ワンオペの構造的上限（希望の提示）
  - 上限70〜80万円
  - 現状は50%→まだ伸びる余地あり

1.3 減少要因の仮説
  - 季節要因（高）
  - 既存顧客離脱（中）
  - 集客チャネル弱体化（中）
```

```
2.3 誤答項目の分析（2問のみ）
  - 炭替え後の対応

2.4 あなたからの質問への回答
```

```markdown
| A) ワンオペ継続 | 70〜75万円 | 0円 | 40〜45万円 | 一人で回したい、利益率重視 |
| B) スタッフ増員 | 90〜100万円 | 約5万円 | 50〜55万円 | 事業拡大したい、休日確保したい |
```

### 5.3 図表作成のガイドライン

#### 表の基本ルール

1. **列の揃え**
   - 文章: 左揃え（`:---`）
   - 数値: 右揃え（`---:`）
   - 評価・ラベル: 中央揃え（`:---:`）

2. **数値の表記**
   - 金額: 「40万円」（単位付き）
   - 割合: 「93%」（%記号）
   - 増減: 「+45%」「-10%」（符号+数値+%）

3. **強調**
   - 重要な数値: `**58万円**`（太字）
   - 最重要項目: 行全体を太字

4. **色の使用は最小限**
   - スコアボードのみ色を許可
   - それ以外は太字・記号で強調

#### ASCIIグラフのルール

1. **軸の明示**
   - Y軸: 単位を明記
   - X軸: 時間軸を明記

2. **ラベル**
   - 重要ポイントに注釈
   - 「現状」「上限」などの基準線

3. **フェーズ分け**
   - └─┘で区間を表現

## 6. デザインルール

### 6.1 見出し階層ルール

```markdown
# 第1章: 現状分析                    ← 章（H1）
## 1.1 売上推移                       ← 節（H2）
### 1.1.1 3ヶ月の推移                 ← 項（H3）
#### ポイント                         ← 補足（H4）※多用しない
```

**ルール**:
- H1（#）: 章タイトルのみ。数字+コロン+タイトル
- H2（##）: 節。章番号.節番号+半角スペース+タイトル
- H3（###）: 項。章.節.項+半角スペース+タイトル
- H4（####）: 補足。タイトルのみ。多用禁止

### 6.2 強調ルール

#### 太字（**）の使用

**使用場面**:
1. 最重要数値: `**93%**`, `**40万円**`
2. 最優先アクション: `**P0: 常連への連絡**`
3. 核心的メッセージ: `**技術力を収益に変換する仕組み**`

**禁止**:
- 見出しを太字にしない（見出し記号で表現）
- 1段落に3箇所以上の太字（散漫になる）

#### 引用（>）の使用

**使用場面**:
1. 重要なメッセージ: `> 「今日できることを、今日やる」`
2. 連絡例文・台本: `> 「〇〇さん、お久しぶりです...」`
3. 注意喚起: `> **重要**: 売上急落時の最優先は...`

#### コードブロック（```）の使用

**使用場面**:
1. ASCIIグラフ
2. 構造図・フロー図
3. 視覚的な枠組み

**禁止**:
- 長文の引用には使わない
- 表には使わない（Markdown表を使用）

### 6.3 表の使用ルール

#### 表の種類と用途

| 表のタイプ | 列数 | 用途 | 例 |
|:---|:---:|:---|:---|
| 2列表 | 2 | 項目と説明 | 用語解説、Q&A |
| 3列表 | 3 | 比較・評価 | スコア、段階評価 |
| 4列表 | 4 | 詳細データ | 売上推移、施策一覧 |
| 5列以上 | 5+ | 多角的比較 | 選択肢比較 |

#### 表作成のルール

1. **見出し行は必須**
2. **列数は最大6列まで**（それ以上は分割）
3. **1行は80文字以内**（スマホ表示を考慮）
4. **数値列は右揃え**
5. **重要行は太字**

### 6.4 余白・セクション区切り

#### 水平線（---）の使用

**使用場面**:
- 章の終わり
- 大きなセクションの区切り
- エグゼクティブサマリーの前後

**禁止**:
- 節・項の区切りには使わない（見出しで十分）
- 連続使用しない

#### 段落間の余白

```markdown
## 見出し
← 1行空白
本文の段落1。
← 1行空白
本文の段落2。
← 1行空白
### 次の見出し
```

**ルール**:
- 見出しの後: 1行空白
- 段落間: 1行空白
- 表・図の前後: 1行空白
- 2行以上の空白は禁止

### 6.5 リスト記号のルール

#### 箇条書き（-）

**使用場面**:
- 並列の項目
- 順序なしの列挙

```markdown
- 項目1
- 項目2
- 項目3
```

#### 番号付きリスト（1.）

**使用場面**:
- 手順・ステップ
- 優先順位
- 時系列

```markdown
1. 第1ステップ
2. 第2ステップ
3. 第3ステップ
```

#### チェックリスト（- [ ]）

**使用場面**:
- アクションプランのタスク
- マイルストーン

```markdown
- [ ] タスク1（未完了）
- [x] タスク2（完了）
```

### 6.6 記号・装飾の統一ルール

#### 使用許可記号

| 記号 | 用途 | 例 |
|:---:|:---|:---|
| ● | 重要ポイント | `●───── ワンオペ上限` |
| ✓ | 正答・OK | `✓ 今すぐやること` |
| ✗ | 誤答・NG | `✗ 誤答` |
| ▲▼ | 増減 | `▲10%`, `▼27%` |
| → | 変化・遷移 | `40万 → 58万` |
| ◯ | 正解マーク | `正解: ◯` |
| ✕ | 不正解マーク | `正解: ✕` |

#### 禁止記号

- 絵文字（😀🎉など）: 専門性を損なう
- 装飾記号（★☆◆◇など）: 散漫になる
- 矢印記号の多用（⇒⇔など）: `→`で統一

### 6.7 色の使用ルール（Markdown外）

**Markdownは色指定不可のため、コメントとして記載**

```markdown
<!-- スコアボード -->
<!-- 技術力93%: 緑色（成功） -->
<!-- 経営力68%: 黄色（注意） -->
```

**色の意味**:
- 緑: 成功・高評価（A評価、90%以上）
- 黄: 注意・中評価（B評価、70〜89%）
- 赤: 危険・低評価（C評価以下、70%未満）

## 7. 付録の位置づけ

### 7.1 付録の役割

**本編**: 意思決定に必要な情報のみ
**付録**: 詳細な根拠・補足説明

### 7.2 付録の構成

```
付録A: 分析計画サマリー（P-hacking防止）
├─ 分析目的
├─ 指標の事前固定
└─ 検証的分析 vs 探索的分析の区別

付録B: 診断問題の全回答解説
├─ シーシャ技術診断（30問）
│   └─ 正答28問、誤答2問の詳細解説
└─ 経営知識診断（25問）
    └─ カテゴリ別の詳細解説

付録C: データ制約と限界の明示
├─ 不足しているデータ
├─ 断定できないこと
└─ 追加データ取得の推奨

付録D: あなたからの質問への詳細回答
├─ パイプの要素に関する詳細
├─ 気温と吸い心地の関係の科学的説明
└─ 味が出ない原因の全パターン
```

### 7.3 本編と付録の振り分け基準

| 項目 | 本編 | 付録 |
|:---|:---:|:---:|
| 診断スコア | ✓ | |
| スコアの計算式 | | ✓ |
| 重要な誤答2問 | ✓ | |
| 全30問の解説 | | ✓ |
| アクションプラン | ✓ | |
| 施策の学術的根拠 | | ✓ |
| データ制約の明示 | ✓ | |
| 統計手法の詳細 | | ✓ |

### 7.4 付録への誘導

**本編での表記例**:

```markdown
※詳細は付録Bを参照

> より詳しい解説は付録D「あなたからの質問への詳細回答」をご覧ください
```

**ルール**:
- 本編では「詳細は付録X」と軽く触れる程度
- 付録への誘導は必要最小限
- 読者が「本編だけで完結する」感覚を維持

## 9. 設計意図の記録

### 9.1 なぜこの構成なのか

**エグゼクティブサマリー優先**:
- 忙しい経営者が3分で全体を把握できるため
- 本編を読む時間がなくても意思決定可能

**技術診断を経営診断より先に**:
- 「技術は高い」という自信を先に与える
- 「経営の課題」を受け入れる心理的準備

**統合診断（第4章）の配置**:
- 技術と経営の両方を読んだ後に統合
- 「仕組みがあれば伸びる」という希望の提示

**アクションプランを後半に**:
- 現状理解→診断→統合→行動の流れ
- 納得した上でアクションを決断

### 9.2 なぜこの図表なのか

**売上推移表に累計減少額**:
- 「21万円失った」という危機感の醸成

**12ヶ月成長曲線**:
- 希望の可視化
- ワンオペ上限を明示し、現実的な期待値を設定

**週別タスクカレンダー**:
- 「今週何をするか」が一目でわかる
- 所要時間を明示し、実行可能性を高める

### 9.3 なぜこのトーンなのか

**「あなた」という2人称**:
- パーソナライズされた診断という印象
- 「一般論ではなく、あなたのための提案」

**数値の具体化**:
- 「売上を伸ばす」ではなく「40万→58万円」
- 行動の成果が測定可能

**希望と厳しさのバランス**:
- 「34%減」という厳しい現実
- 「93%」という強み
- 「58万円到達可能」という希望
