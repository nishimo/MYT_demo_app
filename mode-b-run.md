# mode-b-run.md
## Claude実行テンプレ（シーシャ技術評価 × 経営評価の統合診断 / integrated_tech_management_diagnostic）

> **Note**: このテンプレートを実行すると、`CLAUDE.md` に定義された共通ガードレール（P-hacking禁止ポリシー、出力品質ポリシー、実行プロトコル等）が自動的に適用されます。

### 使い方
1. 下の入力欄を埋める
2. まず「Step 1」だけ投げる（分析計画固定）
3. 次に「Step 2」統合診断の暫定版
4. 最後に「Step 3」提出版整形

---

## 実行モード
- Mode: `integrated_tech_management_diagnostic`

## 目的
技術評価（品質・再現性・提供）と経営評価（売上・粗利・人件費・リピート）を統合して診断し、
「技術改善をどう経営成果に変えるか」を提案する。
P-hackingを避けるため、分析計画を先に固定する。

## Step 1（最初にこれだけ投げる）
以下の案件データを使って、**最終レポートはまだ作らず**、まず次の4点だけ出力してください。
P-hacking防止のため、分析計画を先に固定してください。

1. 分析計画（P-hacking防止）
   - 技術評価軸（定義つき）
   - 経営KPI（結果指標 / 先行指標）
   - 分析期間・比較基準
   - セグメント（スタッフ別 / 時間帯別 / 曜日別 / 必要なら商品別）
   - 外れ値処理ルール
   - 検証的分析 vs 探索的分析の境界
2. データ不足・矛盾・確認事項（重要度順）
3. 現時点の分析可能範囲
4. 追加取得データTOP10（技術評価と経営データの紐づけ優先）

※禁止：探索的に見つけた示唆を確定事実として扱うこと

## Step 2（統合診断の暫定版）
不足データがあっても分析を止めず、**仮説を明示**したうえで統合診断レポートの暫定版を作成してください。
以下を必ず含めてください。

- 検証的分析（事前固定）
- 探索的分析（追加示唆）
- 技術評価（事実ベース）
- 経営評価（事実ベース）
- 統合診断（技術 × 顧客体験 × 経営KPI）
- 技術面の優先改善
- 経営面の優先改善
- 統合施策（技術改善を利益に変える仕組み）
- 教育・評価制度への落とし込み
- 30/60/90日アクションプラン
- 今やらないこと
- レポートの限界

特に重視する観点：
- 技術/運営の観察事実
- 顧客体験への影響仮説
- 行動指標（追加注文/再来店/レビュー等）への影響仮説
- 売上/粗利/人件費効率への影響仮説
- 検証方法（追加で何を測るべきか）

※相関・関連・仮説・因果を区別して記述すること

---

## Step 3（クライアント提出用に整形）
上記の暫定版を、クライアント提出用に再編集してください。要件は以下です。

- 読みやすさ重視（見出し・箇条書き・必要最小限の表）
- 技術評価の点数だけで結論を出さない
- 経営KPIとの接続を明確にする
- エグゼクティブサマリーを先頭に置く
- 優先順位（1〜5）を明記
- 「今やらないこと」を明記
- P-hacking防止の分析計画サマリーを付録または冒頭に記載
- 最後に「本レポートの限界」を記載
